# CPA問題インポートガイド

新しいセッションでOCRテキストの整形・インポートをスムーズに行うための手順書。

## 概要

ユーザーがOCRで読み取った生テキストを貼り付ける → Claude がOCR誤読を修正しフォーマット整形 → API経由でインポート → 重複確認 → 一時ファイル削除

## インポート手順

### 1. OCRテキスト整形

ユーザーが貼り付けた生テキストを以下のフォーマットに整形する。
整形ルールの詳細は `gemini_instructions.md` を参照。

```
[セクションヘッダー]
N.セクション名
ロロロ 問題N 重要度X [年度情報]
問題文（1行にまとめる）
ロロロ 問題N+1 重要度X
問題文...

ロロロ 問題N 正しい／誤り [根拠条文（あれば残す）]
解説文（1行にまとめる）
追加でチェック！
補足説明文（あれば、1行にまとめる）
ロロロ 問題N+1 誤り 「会計基準名」条文番号
解説文...
```

**構造のポイント**:
- 前半＝問題パート、後半＝解答パート（空行1行で区切る）
- セクションヘッダーは問題パート側にのみ記載
- 章の途中からインポートする場合、最初の問題が属するセクションヘッダーを先頭に付ける（パーサーがセッション割当に使うため）

### 2. 一時ファイルに書き出し

```
Write → C:\Users\sashi\Desktop\apps\cpa-exam\tmp_chN.txt
```

章が大きい場合は分割可: `tmp_ch7_q8_20.txt` のように。

### 3. APIでインポート（PowerShell）

**重要**: bash/curlでは日本語が文字化けするため、必ずPowerShellを使う。

```powershell
$text = [System.IO.File]::ReadAllText("C:\Users\sashi\Desktop\apps\cpa-exam\tmp_chN.txt", [System.Text.Encoding]::UTF8)
$body = @{ text = $text; topicId = <topicId> } | ConvertTo-Json -Depth 10
$bytes = [System.Text.Encoding]::UTF8.GetBytes($body)
$response = Invoke-RestMethod -Uri "http://localhost:3020/api/questions/import-text" -Method POST -Body $bytes -ContentType "application/json; charset=utf-8"
$response | ConvertTo-Json -Depth 10
```

成功レスポンス: `{ "imported": N, "errors": [], "total": N }`

### 4. 重複確認（毎回必須）

ユーザーから明示的に「重複があるかもしれませんので逐次確認してください」と指示あり。

```powershell
$r = Invoke-RestMethod -Uri "http://localhost:3020/api/questions?topicId=<topicId>&limit=100" -Method GET
$q = $r.questions
Write-Output "Total: $($q.Count)"
$q | Group-Object { $_.session.name } | ForEach-Object { Write-Output "  $($_.Name): $($_.Count)" }
Write-Output "IDs: $($q | Sort-Object id | ForEach-Object { $_.id })"
```

確認ポイント:
- 合計数が期待値と一致するか
- IDに不自然な重複がないか
- セッション別の問題数が妥当か

### 5. 一時ファイル削除

```powershell
Remove-Item "C:\Users\sashi\Desktop\apps\cpa-exam\tmp_chN.txt" -Confirm:$false
```

### 6. 重複が見つかった場合の削除

```powershell
# 個別削除
Invoke-RestMethod -Uri "http://localhost:3020/api/questions/<id>" -Method DELETE

# 範囲削除（例: ID 24-37）
24..37 | ForEach-Object {
  Invoke-RestMethod -Uri "http://localhost:3020/api/questions/$_" -Method DELETE
}
```

### 解答行の出典情報（根拠条文）

解答行の後ろに付く出典（根拠条文）は**そのまま残す**。パーサーが `sourceReference` フィールドに保存する:
```
ロロロ 問題1 正しい 「企業会計原則」第三 -B
ロロロ 問題4 誤り 「連続意見書 第三」第一 四2
ロロロ 問題18 正しい 「セグメント情報等の開示に関する会計基準」12
```

## パーサーの仕様（route.ts）

- **API**: `POST /api/questions/import-text` body: `{ text: string, topicId: number }`
- **問題検出**: `/^ロロロ\s*問題\s*(\d+)\s+重要度\s*([ABC])/`
- **解答検出**: `/^ロロロ\s*問題\s*(\d+)\s+(正しい|誤り)/` → 後続テキストを `sourceReference` に保存
- **判定解答検出**: `/^ロロロ\s*問題\s*(\d+)\s*$/` → 正しい/誤りなし → 判定問題モード
- **判定項目検出**: `/^([アイウエオカキクケコ])[.．]\s*(.+)/` → ア～コのサブ項目
- **セクション検出**: `/^(\d+)[.．]\s*(.+)/` → Session名として自動作成
- **追加チェック検出**: `/^追加でチェック/`
- **スキップ**: ページ番号（`①-3`等）、章ヘッダー（`第N章`）、単独「問題」行
- **normalizeLine()**: 問題の前のOCRゴミを `ロロロ ` に置換
- **normalizeAnswer()**: `膜り` / `娯り` → `誤り`
- **年度パース**: `H22` → 2010, `R4` → 2022
- **選択肢生成**: `[{text:"正しい", isCorrect:ans}, {text:"誤り", isCorrect:!ans}]`
- **セッション**: セクションヘッダーから名前を取得し、既存なら再利用、なければ新規作成

### 判定問題対応（ア～ケ分割）

判定問題（例: 関連当事者該当判定）を自動的に個別の正誤問題に分割する。

**問題セクション**: `ロロロ 問題N 重要度X` の後に `ア．text` 形式の行が続く場合、各項目を個別の `ParsedQuestion` に分割。問題文は項目テキストのみ（導入文は除外）。

**解答セクション**: `ロロロ 問題N`（正しい/誤りなし）の後に `ア．該当する/該当しない。「source」` 形式の行が続く場合、各項目の該当/非該当を正しい/誤りにマッピング。

**番号付け**: `親問題番号 * 100 + サブインデックス`（例: 問題14ア→1401, イ→1402）。内部マッチング用であり、DB保存後は通常のID連番になる。

**判定マッピング**: 2パターン対応
- `該当する` → `isCorrect: true`、`該当しない` → `isCorrect: false`
- `対象取引` → `isCorrect: true`、`対象外取引` → `isCorrect: false`

**sourceReference**: 解答行の判定語（`該当する/しない。` or `対象外?取引`）の後に続くテキスト（例: `「関連当事者の開示に関する会計基準」5(3)①`）を保存。

**整形フォーマット**:
```
ロロロ 問題N 重要度X [年度情報]
ア．項目テキスト1
イ．項目テキスト2
...

ロロロ 問題N
ア．関連当事者に該当する。「source」
解説文
イ．関連当事者に該当しない。
解説文
```

## 制約事項

- `npm run build`, `npm install` 等をローカルで実行しない（Docker内で完結）
- bash/curl は日本語が文字化けするため API呼び出しには PowerShell を使う
- インポートごとに重複確認を必ず行う
- 問題文・解説文の内容は変更しない（OCR誤読修正のみ）
