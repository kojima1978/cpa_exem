# CPA問題テキスト整形 指示書

## 目的

OCRで読み取ったCPA短答式問題のテキストを、アプリにインポートできるJSON形式に整形してください。

## 出力フォーマット（厳守）

**JSON配列**のみを出力してください。余計な説明・コメント・マークダウンのコードフェンスは不要です。

```json
[
  {
    "num": 1,
    "section": "真実性の原則",
    "difficulty": "B",
    "year": null,
    "text": "問題文（OCR修正済み・1行）",
    "answer": "正しい",
    "sourceReference": "「企業会計原則」第一 一",
    "explanation": "解説文（OCR修正済み・1行）",
    "additional": ""
  }
]
```

### フィールド定義

| フィールド | 型 | 説明 |
|---|---|---|
| `num` | number | 問題番号（原文のまま。章ごとに1から始まる） |
| `label` | string\|省略 | 判定問題のサブ項目ラベル（"ア","イ",...）。通常問題では省略 |
| `section` | string | セクション名（例: "真実性の原則"）。原文の `N.セクション名` から取得 |
| `difficulty` | string | "A"（難）/ "B"（標準）/ "C"（易） |
| `year` | string\|null | 最初の年度コード（"H25", "R4" 等）。なければ `null` |
| `text` | string | 問題文（OCR修正済み・改行結合済み・1行） |
| `answer` | string | "正しい" または "誤り" |
| `sourceReference` | string | 根拠条文（例: `「企業会計原則」第一 一`）。なければ空文字列 `""` |
| `explanation` | string | 解説文（OCR修正済み・改行結合済み・1行）。なければ空文字列 `""` |
| `additional` | string | 「追加でチェック！」の補足説明文（1行）。なければ空文字列 `""` |

### 判定問題（ア～ケ形式）

「該当するか否か判定しなさい」のような問題は、各サブ項目を**個別の正誤問題に分割**する。

**処理手順**:
1. 導入文（例: 「次のア～ケについて，関連当事者に該当するか否か判定しなさい。」）から判定基準を抽出
2. 各項目テキストに判定基準の述語を補完して完全な文にする（例: 「財務諸表作成会社の親会社」→「財務諸表作成会社の親会社は，関連当事者に該当する。」）
3. 解答の「該当する」→ `"正しい"`、「該当しない」→ `"誤り"` にマッピング
4. `label` フィールドにサブ項目ラベルを記載

入力:
```
nnn 問題14 重要度B H25過去間類題
次のアーケについて，関連当事者に骸当するか否か判定しなさい。

イ．財務諸表作成会社の子会社

イ．関連当事者に該当する 「関連当事者の開示に関する会計基準」5(3)②
解説文...
```

- 解答がサブ項目ごとに「該当する」「該当しない」で記載されている

## 整形ルール

### 1. 構造の並べ替え

- 原文ではページごとに問題と解答が交互に出てくるが、**問題と解答を対応付けて1つのJSONオブジェクトにまとめる**
- 問題番号は原文のまま維持（章ごとに1から始まる）
- セクションヘッダー（`N.セクション名`）は `section` フィールドに反映する。以降の問題は次のセクションヘッダーが現れるまで同じ `section` 値を使う

### 2. 問題行の認識

形式: `ロロロ 問題N 重要度X [年度情報]`

- `ロロロ` の前のOCRゴミ（nnn, DDD, DD口 等）は無視して認識する
- 重要度は `A` / `B` / `C`（小文字の場合は大文字に変換）
- 年度情報の取得:
  - 平成: `H` + 数字（例: H22, H25）
  - 令和: `R` + 数字（例: R4, R6）
  - 複数年度がある場合は最も古いものを `year` フィールドに設定
  - `過去問` / `過去問類題` の区別は不要（年度コードのみ抽出）

### 3. 解答行の認識

形式: `ロロロ 問題N 正しい [根拠条文]` または `ロロロ 問題N 誤り [根拠条文]`

- 「正しい」「誤り」の後ろに続く出典情報 → `sourceReference` にそのまま格納
- 出典の例: `「企業会計原則」第一 一`、`「連続意見書 第三」第一 四2`、`「セグメント情報等の開示に関する会計基準」12`

### 4. 補足（追加でチェック）

- `追加でチェック！` で始まる行の後に続く補足テキスト → `additional` に格納
- OCR誤読の修正: `追加でチェック!`（半角!）, `追加でチェック1`, `追加でチェックI` → すべて `追加でチェック！`
- `追加でチェック！` の行に付いている年度情報（例: `H28過去問`）は除去

### 5. テキスト整形

- **改行の結合**: OCRで途中改行されている文は1行にまとめる
- **句読点の統一**: `，`（全角カンマ）と `。`（全角句点）を使用。読点 `、` は `，` に変換
- **括弧の統一**: 半角 `(` `)` → 全角 `（` `）`（ただし根拠条文内の `5(3)①` のような条文番号は半角のまま）
- **不要要素の除去**:
  - ページ番号行（`①-3`, `②-1` 等の丸数字形式）
  - 章ヘッダー（`第N章 ...`）
  - 単独の「問題」行
  - 重複するセクションヘッダー（ページ区切りで再出現するもの）

### 6. やってはいけないこと

- 問題文や解説文の**内容を変更・要約・省略しない**（OCR誤読の修正のみ行う）
- 問題番号を勝手に振り直さない
- 存在しない問題や解答を追加しない
- 重要度を変更しない
- 自分の判断で「正しい」「誤り」を変更しない
- 根拠条文（sourceReference）を編集・省略しない

## 入力テキストの構造について

入力は以下のような構造になっています:

1. 各ページの先頭に丸数字ページ番号（`①-1`、`②-3` 等）
2. 左ページ（奇数）に問題、右ページ（偶数）に解答
3. ページごとに章名ヘッダー（`2．一般原則` 等）が繰り返し出現
4. 問題と解答が交互に配置されている

これらを**問題と解答を対応付けてJSONオブジェクトにまとめて**ください。

## チェックリスト

出力前に以下を確認してください:

- [ ] 出力がJSON配列として構文的に正しいか
- [ ] すべての問題に対応する解答があるか
- [ ] 問題番号が原文通りか（振り直していないか）
- [ ] OCR誤読が修正されているか
- [ ] 改行が適切に結合されて1行になっているか
- [ ] `sourceReference` が原文通り保持されているか
- [ ] 判定問題は個別のオブジェクトに分割されているか
- [ ] `answer` が「正しい」または「誤り」のみか
- [ ] `additional` に追加チェックの内容が正しく格納されているか
